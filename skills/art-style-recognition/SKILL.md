# 艺术人文风格标签识别

分析服装图片，识别其艺术人文风格标签。支持41种预定义风格分类。

## 输入

- **image_url**（必填）：服装图片URL
- **task_type**（默认 `art_tag`）：任务类型标识

## 工作流程概览

```
图片URL → HTTP获取图片 → 生成密集风格描述 → 知识检索相似内容
                                                    ↓
                                        [有检索结果?]
                                       /            \
                                     是              否
                                     ↓               ↓
                              融合机制判定      直接LLM风格识别
                             /          \            ↓
                     [满足条件]    [不满足]     输出风格标签
                        ↓            ↓
                   直接输出     LLM风格识别
                                    ↓
                              输出风格标签
```

---

## Step 1: 生成密集风格描述

使用视觉模型（Gemini 3 Flash）对图片进行深度分析，生成结构化的风格描述文本。

**System Prompt（完整版见 `references/image-description-prompt.md`）：**

核心要求：
- 角色：资深时尚符号学分析师和计算机视觉专家
- 任务：为向量数据库检索系统生成"密集风格描述"
- 分析维度：气质与氛围、色彩与图案、轮廓与面料、设计细节、环境语境、文化溯源

**输出格式：**
```
气质氛围：[氛围感], [气质], [态度], [年龄感], [环境滤镜感], [穿搭思路]
| 文化溯源: [起源年代/背景], [核心地域/亚文化], [艺术/哲学流派]
| 视觉属性: [色调色彩], [面料肌理], [版型廓形], [造型逻辑], [关键细节], [图案], [环境氛围/灯光]
```

**约束：**
- 杜绝填充词（"图片显示"、"我能看到"等）
- 高密度使用具体时尚术语
- 只关注有助于特定时尚风格的视觉元素
- 文化溯源：有明显时尚文化基因时才输出，否则输出"无"

---

## Step 2: 知识检索（向量检索）

使用 Step 1 生成的文本描述作为查询，进行向量相似度检索：
- 嵌入模型：text-embedding-3-large
- 检索方式：纯向量检索（权重1.0）
- 相似度阈值：0.8
- 返回 Top 5 结果

每个检索结果包含：文本内容（text）、相似度分数（score）、位置（position）

---

## Step 3: 融合机制

根据检索结果决定输出策略：

### 3a. 无检索结果 → 直接进入 Step 4（LLM风格识别）

### 3b. 有检索结果 → 执行融合判定

对检索到的 Top 5 结果，计算每条结果文本的 MD5，查询历史数据库获取其对应的已标注风格标签，然后执行以下判定逻辑：

```python
# 条件1：Top 1 相似度得分 >= 0.9，直接输出该标签
if top1_score >= 0.9:
    return top1_label

# 条件2：存在2条以上召回时，某标签出现频率 >= 60%，输出该标签
label_counts = Counter(all_labels)
top_label, top_count = label_counts.most_common(1)[0]
if top_count / total >= 0.6:
    return top_label

# 条件3：以上条件均不满足 → 进入 Step 4（LLM风格识别）
```

---

## Step 4: 基于标签定义的LLM风格识别

当融合机制无法决定时，使用 LLM 结合完整的风格定义进行识别。

### 4a. 无检索结果时的识别（使用完整内嵌定义）

使用 Gemini 3 Flash 模型，system prompt 包含完整的41种风格定义体系。

**完整风格定义见 `references/style-definitions.md`**

**核心识别逻辑：**
1. 先对风格定义进行精准理解，掌握风格特征及风格间的核心差异
2. 对图片进行充分深刻且专业的洞察理解（气质氛围、文化溯源、视觉属性）
3. 找与图片最契合的2个风格标签
4. 聚焦气质氛围和文化溯源，从2个中选出最适配的1个
5. 输出中文风格名称和分析依据（100字内）

**关键判断优先级：**
1. 优先判断图片的气质氛围与哪个风格最契合（最重要）
2. 辩证分析风格下易混淆风格的关键特征
3. 注意风格定义里排除项内容
4. 注意图片造型里的细节是否符合风格核心定义

**输出格式（JSON）：**
```json
{
  "label": "风格中文名称",
  "reason": "分析依据，100字以内"
}
```

### 4b. 有检索结果但融合不满足条件时

使用 GPT-5 Mini 模型，从数据库查询的标签定义中进行匹配：

```
<role>你是从业10年的女装服装专家，精通人文艺术风格</role>
<goal>对视觉输入的图片进行识别，根据 definition 里面的风格标签定义，找出最匹配的标签。</goal>
<definition>{从数据库查询的标签定义}</definition>
<output>以json格式进行输出：{style: "xxx"}</output>
```

## 输出

| 字段 | 说明 |
|------|------|
| **image_url** | 原始图片URL |
| **task_type** | 任务类型 |
| **image_desc** | 图片的密集风格描述文本 |
| **style_label** | 识别出的风格标签（中文） |
| **reason** | 识别依据说明 |
| **result_md5** | 描述文本的MD5哈希（用于去重和缓存） |
| **similarity_score** | 与知识库最相似内容的得分（0-1，融合机制输出时有值） |

## 41种风格体系

完整定义见 `references/style-definitions.md`，按以下体系组织：

- **A. 经典与正装体系**：经典正装、常春藤学院、小香风、静奢风、权力套装
- **B. 制服与功能体系**：工装风、军旅风、海军风、JK风、专业运动、专业户外
- **C. 城市休闲与街头体系**：嘻哈风、滑板风、西部丹宁、机车骑士、美式复古、户外山系、松弛运动、Y2K、Mob Wife
- **D. 音乐与青年亚文化体系**：摇滚朋克、暗黑哥特、格兰奇、Indie Sleaze、亚比女团风
- **E. 艺术思潮与设计观念体系**：极简主义、解构主义、前卫黑系、波普艺术
- **F. 科幻与未来叙事体系**：机能风、未来主义、废土风
- **G. 浪漫与历史复兴体系**：洛丽塔、芭蕾风、宫廷风、波西米亚、田园风
- **H. 地域与文化生活方式体系**：南法度假、热带度假、新中式
- **其他**：不匹配以上任何风格时选择

每种风格定义包含：风格概述、气质氛围、文化溯源、易混淆风格（关键区别）、排除项
