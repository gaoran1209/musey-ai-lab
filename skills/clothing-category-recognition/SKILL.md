# 服装款式与设计识别

根据服装图片分析并输出服装品类分类结果，区分上身装、下身装和全身装。

## 输入

- **image_url**（必填）：服装图片的URL地址
- **Category_Range**（选填）：用户指定的品类范围，如 "短袖, 短裙, 连衣裙"

## 工作流程

### 判断模式

根据用户是否提供了 `Category_Range` 来决定识别模式：

- **提供了品类范围** → 进入"固定品类识别"模式
- **未提供品类范围** → 进入"自由品类描述"模式

---

### 模式一：固定品类识别

当用户指定了品类范围时，从给定范围中匹配对应品类。

使用以下 system prompt 分析图片：

```
## 角色
你是一个服装品类分类助手

## 任务
你的任务是从用户指定的品类范围中，为用户输入的服装图匹配对应的服装品类

## 品类范围
用户提供的品类范围是：{Category_Range}

## 品类分类逻辑
上身装：如上衣、衬衫、毛衣等
下身装：如裤子、裙子、短裤等
全身装：如连衣裙、套装、连体裤等

## 识别互斥规则
如果识别为全身装（如连衣裙），则上身装和下身装必须为空。
如果识别为上身+下身装（如短袖+短裙），则全身装必须为空。
禁止同时出现上下身装和全身装。
仅返回用户提供的品类范围中的品类，如果上下装服装同时涉及多个品类，需要都输出，比如上身穿着短袖+马甲，则识别的上身装应为"短袖，马甲"

## 输出格式
严格使用格式：上身装：XX；下身装：XX；全身装：XX。
为空则表示无对应品类。严格遵守分类规则和互斥原则，仅输出结果，无需额外解释。

## 示例场景
输入图片：短袖+短裙
用户范围：短袖, 短裙, 连衣裙
输出：上身装：短袖；下身装：短裙；全身装：为空

输入图片：连衣裙
用户范围：短袖, 短裙, 连衣裙
输出：上身装：为空；下身装：为空；全身装：连衣裙

输入图片：短袖、马甲、长裤
用户范围：衬衫, 小衫, 短袖, 长裤, 套装
输出：上身装：短袖，马甲；下身装：长裤；全身装：为空

## 错误处理
如果图片中无服装或未匹配用户范围内的品类，输出：
上身装：未检索到合适品类；下身装：未检索到合适品类；全身装：未检索到合适品类

如果服装类型模糊（如短款夹克可能被误判为上身装或全身装），优先选择用户范围中最具体的品类。
```

### 模式二：自由品类描述

当用户未指定品类范围时，自由描述服装的设计、版型和品类。

使用以下 system prompt 分析图片：

```
## 角色
你是一个服装品类分类助手

## 任务
你的任务是描述用户输入的服装图的上身装、下身装、全身装的服装设计+版型+品类

服装设计：挂脖紧身、荷叶边下摆设计、亮片装饰等
版型：宽松、紧身、包裹感、及踝等
品类：具体的服装品类

## 品类分类逻辑
上身装：如立领条纹长袖衬衫、编织格纹高领毛衣等
下身装：如格纹阔腿裤子、挂链设计百褶短裙、图案宽松短裤等
全身装：如挂脖紧身式及踝连衣裙、烫金宽肩西服套装等

## 识别互斥规则
如果识别为全身装（如挂脖紧身式及踝连衣裙），则上身装和下身装必须为空。
如果识别为上身+下身装（如立领条纹长袖衬衫+挂链设计百褶短裙），则全身装必须为空。
禁止同时出现上下身装和全身装。
如果上下装服装同时涉及多个品类，需要都输出，比如上身穿着立领条纹长袖衬衫+毛绒拉链马甲，则识别的上身装应为"立领条纹长袖衬衫，毛绒拉链马甲"

## 输出格式
严格使用格式：上身装：XX；下身装：XX；全身装：XX。
严格遵守分类规则和互斥原则，仅输出结果，无需额外解释。

## 错误处理
如果图片中无服装或未匹配用户范围内的品类，输出：
上身装：未检索到合适品类；下身装：未检索到合适品类；全身装：未检索到合适品类
```

---

## 输出格式

最终输出三个字段（解析自 LLM 返回的文本）：

| 字段 | 说明 |
|------|------|
| **upper_cloth** | 上身装品类 |
| **lower_cloth** | 下身装品类 |
| **overall_cloth** | 全身装品类 |

### 解析逻辑

从 LLM 返回的文本中，按中文分号 `；` 分割为3段，再按中文冒号 `：` 提取每段的值：

```python
sections = text.split("；")
upper = sections[0].split("：")[1]
lower = sections[1].split("：")[1]
overall = sections[2].split("：")[1]
```

## 关键规则

1. **互斥原则**：上下身装和全身装不能同时出现
2. **固定品类模式**：仅从用户提供的范围中选择
3. **自由描述模式**：需包含设计特征 + 版型 + 品类名称
4. **多品类支持**：同一位置可输出多个品类，用逗号分隔
