欢迎使用 Musey AI Lab

Musey AI Lab 是一个基于 Gemini 驱动的可视化 AI 画布，专为创意图像生成、编辑和分析工作流而设计。通过拖拽操作，在无限画布上自由搭建节点式 AI 流水线。

你可以做什么？

  • 图像生成 — 描述任意场景，Gemini 将文字转化为图像
  • 图像编辑 — 更换背景、替换模特、虚拟试穿服装
  • 视频生成 — 使用 Veo 将图像或提示词转为短视频
  • 风格分析 — 从图像中识别艺术风格和服装品类
  • 工作流串联 — 连接节点，将上游结果传递给下游操作

快速开始

在本地运行 Musey AI Lab，几分钟内生成你的第一张 AI 图像。

前置条件

  • Node.js ≥ 20.0.0
  • 一个有效的 Google Gemini API Key（点此获取）

安装步骤

克隆仓库

git clone <repo-url>
cd musey-ai-lab

安装依赖

npm install

启动开发服务器

npm run dev

在浏览器中打开 http://localhost:3000。

配置 API Key

点击左侧栏底部的头像图标，输入你的 Gemini API Key，然后点击保存。密钥仅存储在浏览器本地，不会发送至任何第三方服务器。

生成你的第一张图

创建节点

在画布空白处双击，或使用左侧栏的 + 按钮添加一个图像节点。

编写提示词

生成图像

点击生成按钮，图像将在几秒钟内出现在节点中。

你还可以点击提示词区域的麦克风图标，使用语音输入。

核心概念

了解 Musey AI Lab 的基本构建模块：无限画布、节点、连线，以及数据如何在它们之间流动。

无限画布

画布是一个无限延展的 2D 空间，你可以在上面自由搭建 AI 工作流。拖拽背景可以平移视图，滚轮可以缩放，节点可以自由排列。

节点

节点是画布的核心单元。每个节点代表一个操作或一段内容：

  • 图像节点 — 使用 AI 生成、编辑和分析图像。支持文字提示词、参考图和技能。
  • 视频节点 — 使用 Veo 模型将提示词或图像生成短视频。

节点可以移动、调整大小、相互连接。每个节点左侧有输入端口，右侧有输出端口。

连线（Edges）

连线是节点之间的连接。将一个图像节点的输出连接到另一个节点的输入后，生成的图像将自动成为下游节点的参考图。

这使得以下工作流成为可能：

  • 生成图像 → 更换背景
  • 生成人物 → 虚拟试穿不同服装
  • 生成图像 → 转化为视频动画

参考图

当节点通过连线相连后，上游图像会出现在下游节点的参考列表中。你可以在提示词中使用 @ 语法来引用特定的参考图像，告诉 AI 具体使用哪张图以及如何使用。

技能

技能是预构建的 AI 操作，超越了简单的文生图能力。它们使用专门的多步提示链来完成复杂的编辑任务，如更换背景、替换模特和虚拟试穿。

所有节点状态（位置、连接、图像）存储在浏览器内存中。刷新页面将重置画布。

视频节点

在画布上直接使用 Veo 模型生成 AI 短视频。

创建视频节点

使用左侧栏的 + 菜单，选择"视频节点"。你也可以将图像节点连接到视频节点，实现图生视频。

视频生成

视频节点使用 Veo 3.1 模型生成视频。提供一段文字提示词，描述你想要的运动效果和场景。

视频生成是异步处理的，可能需要 30 秒到几分钟不等。进度指示器会显示当前状态。

视频控件

  • 播放 / 暂停 — 切换视频播放状态
  • 进度条 — 拖动到视频的任意位置
  • 下载 — 将视频文件保存到本地
  • 删除 — 移除视频

图生视频

将图像节点的输出连接到视频节点的输入。生成的图像将作为第一帧，Veo 模型会根据你的提示词对其进行动画处理。

连接与引用

了解如何将节点连接在一起，搭建多步骤 AI 工作流。

创建连接

从一个节点的输出端口（右侧）拖拽到另一个节点的输入端口（左侧）即可创建连接。上游节点的生成图像会自动成为下游节点的参考图。

使用参考图

连接建立后，参考图像会出现在下游节点的参考面板中。你可以：

  • 隐式使用 — AI 会自动将连接的图像作为上下文参考
  • 在提示词中使用 @引用名称 显式引用特定图像

历史记录

在历史面板中追踪所有生成请求、结果和错误信息。

历史面板

从左侧栏打开历史面板。每条记录包含：

  • 提示词 — 使用的文字提示词
  • 状态 — 成功、失败或进行中
  • 耗时 — 生成所花费的时间
  • 时间戳 — 请求发起的时间
  • 错误详情 — 如果生成失败，显示错误信息

数据持久化

历史记录存储在浏览器的 localStorage 中，键名为 muse-history。它们在浏览器会话之间持久保留，但仅限于你的本地设备。

清除浏览器数据将删除所有历史记录，目前没有云端备份。

技能工作原理

技能使用多步流水线：

  1. 特征提取 — 分析输入图像，理解内容和上下文
  2. 提示词构建 — 用提取的特征构建专业提示词
  3. 生成 / 分析 — 使用构建好的提示词执行 AI 模型
  4. 结果呈现 — 在节点中展示输出，或创建新的连接节点

更换背景

替换或融合图像背景，完美保留主体人物。

模式

氛围融合

从原始图像中提取背景特征，然后通过文字引导重新生成，创造协调的氛围转换。适合改变场景的情绪或环境，同时保持自然感。

  • 使用 gemini-3.1-pro-preview 进行场景提取
  • 产出自然、协调的结果
  • 示例：将白天的照片变为夕阳场景

精准替换

直接的双图融合，以精准的方式替换背景。适合你心中已有明确背景画面的场景。

  • 提供新背景的文字描述
  • 主体边缘保持清晰锐利
  • 示例：将模特置于特定的摄影棚或场景中

使用方法

在节点中选择一张图片，或确保节点已有生成的图像

点击更换背景技能按钮

选择模式（氛围融合或精准替换）

输入背景描述并生成

更换模特

替换图像中的人物或面部，同时保留服装、姿态和场景。

模式

换脸模式

直接替换面部 — 仅更换面部，保持身体、服装和背景不变。适合快速换脸。

复刻模式

更高级的流水线：先提取场景特征和时尚细节，再与目标面部重新组合。对全身照的处理效果更自然、更连贯。

  • 将场景和时尚特征提取为结构化 JSON
  • 动态注入到生成提示词中
  • 更好地保留服装和姿态细节

使用条件

  • 至少需要两张图片：原始场景图和目标人脸图
  • 将两个图像节点都连接到技能节点

虚拟试穿

将服装单品应用到人物图像上，实现逼真的穿搭效果，精准还原颜色和透视。

工作原理

提供一张人物基底图和一张或多张服装图片。AI 会将服装穿到人物身上，同时保持姿态、体型和背景不变。

支持的服装类型

类别示例
上装T恤、衬衫、夹克、毛衣、外套
下装裤子、裙子、短裤、牛仔裤
连体装连衣裙、连体裤、紧身衣
配饰鞋子、包包、帽子

多件服装支持

你可以同时应用多件服装。每件服装图片都会标注其品类标签（上装、裤子、鞋子等），以确保精准放置。

为获得最佳效果，建议使用白色或透明背景的干净商品图。

服装品类识别

自动识别和分类图像中的服装单品。

模式

固定范围

在用户提供的预定义品类集合中进行匹配。适合你已经知道可能品类的场景。

自由描述

开放式分析，描述每件服装的设计、轮廓和类型，不受预定义约束。

输出格式

结果以结构化 JSON 返回，包含三个字段：

{
  "upper_cloth": "宽松牛仔夹克",
  "lower_cloth": "高腰阔腿裤",
  "overall_cloth": null
}
互斥规则：如果检测到 overall_cloth（如连衣裙），则 upper_cloth 和 lower_cloth 为 null，反之亦然。

风格识别

从图像中识别艺术和时尚风格标签，覆盖 8 大分类共 41 种预定义风格。

风格分类

系统可识别 8 大类风格，包括但不限于：

  • 经典 — 极简主义、学院风、商务休闲
  • 街头 — 街头潮流、Grunge、嘻哈
  • 浪漫 — 田园风、波西米亚、复古
  • 先锋 — 解构主义、未来感、艺术风
  • 民族 — 民族风、传统复兴
  • 运动 — 运动休闲、山系穿搭、机能风
  • 奢华 — 老钱风、静奢风
  • 亚文化 — 朋克、哥特、Y2K

识别流程

  1. 密集描述 — 使用 Gemini 生成详细的风格描述
  2. 向量检索 — 从 41 种风格定义中找到最接近的匹配
  3. 分数融合 — 应用阈值过滤
  4. LLM 兜底 — 如果没有明确匹配，使用 LLM 推理

输出

结果包含匹配的风格标签和简要说明：

{
  "label": "田园风 Cottagecore",
  "reason": "碎花连衣裙搭配泡泡袖，草帽和自然户外场景，呈现浪漫化的乡村美学。"
}

常见问题

关于 Musey AI Lab 的常见疑问解答。

基本问题

我的 API Key 安全吗？

安全。你的 API Key 仅存储在浏览器的 localStorage 中，直接发送到 Google 的 API 端点，不会经过任何第三方服务器。

这个应用需要后端吗？

不需要。Musey AI Lab 是一个纯前端应用，所有 AI 调用都从浏览器直接发往 Google 的 API。

刷新页面后画布会保存吗？

目前画布状态（节点、连接、图像）仅存储在内存中。刷新页面会重置画布。历史记录会持久保存在 localStorage 中。

生成相关

为什么我的生成失败了？

常见原因：

  • API Key 无效 — 在左侧栏设置中检查你的密钥
  • 配额超限 — 可能已达到 API 调用频率限制
  • 内容策略 — 提示词可能触发了安全过滤器

为什么视频生成这么慢？

Veo 的视频生成计算量很大，在 Google 的服务器上异步运行。根据复杂度不同，通常需要 30 秒到几分钟。

可以一次生成多张图片吗？

可以。在图像节点中设置"数量"参数，每次请求可生成 1-4 张图片。

技能相关

为什么更换模特需要两张图片？

该技能需要原始场景图像（包含要替换的人物）和目标人脸/人物的参考图像。

支持哪些图片格式？

支持导入 PNG、JPEG 和 WebP 格式。生成的图像以 PNG 格式返回。

部署相关

如何部署到生产环境？

运行 npm run build 在 /dist 目录生成生产构建。将其内容部署到任意静态托管服务（Vercel、Netlify、GitHub Pages 等）。

不要在构建时将 API Key 嵌入环境变量。当前的用户输入模式是前端应用的安全做法。
