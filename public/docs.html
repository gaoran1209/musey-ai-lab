Quick Start

Get Musey AI Lab running on your machine and generate your first image in minutes.

Prerequisites

  • Node.js ≥ 20.0.0
  • A valid Google Gemini API Key (get one here)

Installation

Clone the repository

git clone <repo-url>
cd musey-ai-lab

Install dependencies

npm install

Start the dev server

npm run dev

Open http://localhost:3000 in your browser.

Configure your API Key

Click the avatar icon in the bottom-left sidebar, enter your Gemini API Key, and press save. The key is stored locally in your browser — it is never sent to any third-party server.

Your First Image

Create a node

Double-click anywhere on the canvas, or use the + button in the sidebar to add an Image Node.

Write a prompt

Type a description of the image you want to create, e.g. A serene Japanese garden at sunset.

Generate

Click the Generate button. Your image will appear in the node within seconds.

You can also use voice input by clicking the microphone icon in the prompt area.

Core Concepts

Understanding the building blocks of Musey AI Lab: the infinite canvas, nodes, edges, and how data flows between them.

The Infinite Canvas

The canvas is an infinite 2D space where you build your AI workflows. You can pan by dragging the background, zoom with the scroll wheel, and arrange nodes freely in any layout.

Nodes

Nodes are the core units of the canvas. Each node represents a single operation or piece of content:

  • Image Node — Generate, edit, and analyze images using AI. Supports text prompts, reference images, and skills.
  • Video Node — Generate short videos from prompts or images using the Veo model.

Nodes can be moved, resized, and connected to each other. Each node has input handles (left side) and output handles (right side).

Edges (Connections)

Edges are the lines that connect nodes. When you connect an Image Node's output to another node's input, the generated image becomes available as a reference image in the downstream node.

This enables powerful workflows like:

  • Generate an image → change its background
  • Generate a person → try on different clothes
  • Generate an image → animate it into a video

Reference Images

When nodes are connected, the upstream image appears in the downstream node's reference list. You can mention references in your prompt using the @ syntax to tell the AI exactly which image to use and how.

Skills

Skills are pre-built AI operations that go beyond simple text-to-image generation. They use specialized prompt chains to achieve complex editing tasks like background replacement, model swapping, and virtual try-on.

All node state (positions, connections, images) is stored in your browser's memory. Refreshing the page will reset the canvas.

Video Node

Generate short AI-powered videos using the Veo model, directly on the canvas.

Creating a Video Node

Use the sidebar's + menu and select "Video Node". You can also convert an image into a video by connecting an Image Node to a Video Node.

Video Generation

The Video Node uses the Veo 3.1 model to generate videos. Provide a text prompt describing the motion and scene you want.

Video generation is asynchronous — it may take 30 seconds to a few minutes. A progress indicator shows the current status.

Video Controls

  • Play / Pause — toggle video playback
  • Progress bar — seek to any point in the video
  • Download — save the video file locally
  • Delete — remove the video

Image-to-Video

Connect an Image Node's output to a Video Node's input. The generated image will be used as the first frame, and the Veo model will animate it based on your prompt.

Connections & References

Learn how to link nodes together to build multi-step AI workflows.

Creating Connections

Drag from a node's output handle (right side) to another node's input handle (left side) to create a connection. The upstream node's generated images automatically become available as reference images in the downstream node.

Using References

Once connected, reference images appear in the downstream node's reference panel. You can:

  • Use them implicitly — the AI will consider connected images as context
  • Use @reference-name in your prompt to explicitly reference a specific image

Workflow Examples

History

Track all your generation requests, results, and errors in the history panel.

History Panel

Open the history panel from the left sidebar. Each record shows:

  • Prompt — the text prompt used
  • Status — success, error, or in-progress
  • Duration — how long the generation took
  • Timestamp — when the request was made
  • Error details — if generation failed, the error message

Persistence

History records are stored in your browser's localStorage under the key muse-history. They persist across browser sessions but are local to your device.

Clearing your browser data will erase all history records. There is no cloud backup.

Change Background

Replace or blend the background of an image while perfectly preserving the subject.

Modes

Atmosphere Blend

Extracts background features from the original image, then regenerates with text guidance to create a harmonious atmosphere shift. Best for changing the mood or setting of a scene while keeping it natural.

  • Uses gemini-3.1-pro-preview for scene extraction
  • Produces natural, cohesive results
  • Example: Change a daytime photo to a sunset scene

Precise Replace

Direct dual-image fusion that replaces the background with surgical precision. Best when you have a specific background in mind.

  • Provide a text description of the new background
  • Keeps subject edges sharp and clean
  • Example: Place a model in a specific studio or location

Usage

Select an image in a node, or ensure the node has a generated image

Click the Change Background skill button

Choose the mode (Atmosphere Blend or Precise Replace)

Enter your background description and generate

Change Model

Replace the person or face in an image while preserving clothing, pose, and scene.

Modes

Face Swap

Direct facial replacement — swaps only the face while keeping everything else (body, clothes, background) intact. Best for quick face changes.

Replica Mode

A more advanced pipeline that extracts scene features and fashion details, then recombines them with the target face. Produces more natural and coherent results for full-body shots.

  • Extracts scene + fashion features as structured JSON
  • Dynamically injects into generation prompt
  • Better for preserving outfit and pose details

Requirements

  • At least two images: the original scene and the target face
  • Connect both image nodes to the skill node

Virtual Try-On

Apply clothing items to a person image with realistic fit, perspective, and color accuracy.

How It Works

Provide a base image of a person and one or more clothing images. The AI will dress the person in the provided garments while preserving their pose, body shape, and background.

Multi-Garment Support

You can apply multiple clothing items simultaneously. Each garment image is tagged with its category (top, pants, shoes, etc.) for accurate placement.

For best results, use clean product photos of clothing items on a white or transparent background.

Clothing Category Recognition

Automatically identify and classify clothing items in an image.

Modes

Fixed Range

Match clothing against a predefined set of categories provided by the user. Useful when you know the possible categories.

Free Form

Open-ended analysis that describes the design, silhouette, and type of each clothing item without predefined constraints.

Output Format

Results are returned as structured JSON with three fields:

{
  "upper_cloth": "Oversized denim jacket",
  "lower_cloth": "High-waisted wide-leg pants",
  "overall_cloth": null
}
Mutual exclusivity rule: If overall_cloth is detected (e.g., a dress), upper_cloth and lower_cloth will be null, and vice versa.

Art Style Recognition

Identify artistic and fashion style labels from images across 41 predefined categories in 8 style groups.

Style Categories

The system recognizes styles across 8 major groups, including but not limited to:

  • Classic — Minimalist, Preppy, Business Casual
  • Street — Streetwear, Grunge, Hip-Hop
  • Romantic — Cottagecore, Bohemian, Vintage
  • Avant-Garde — Deconstructed, Futuristic, Art-Inspired
  • Cultural — Ethnic, Traditional Revival
  • Sporty — Athleisure, Gorpcore, Techwear
  • Luxury — Old Money, Quiet Luxury
  • Subculture — Punk, Gothic, Y2K

Recognition Pipeline

  1. Dense description — Generate a detailed style description using Gemini
  2. Vector search — Find closest matches from the 41 style definitions
  3. Score fusion — Apply threshold-based filtering
  4. LLM fallback — If no clear match, use LLM reasoning

Output

Results include the matched style label and a brief explanation of why it was chosen:

{
  "label": "Cottagecore",
  "reason": "Floral print dress with puff sleeves, straw hat, and natural outdoor setting suggest a romanticized rural aesthetic."
}

Configuration

Customize your Musey AI Lab setup.

API Key

Your Gemini API key is stored in localStorage under the key gemini_api_key. It is never sent to any server other than Google's API endpoints.

To update your key:

  1. Click the avatar icon in the sidebar (bottom-left)
  2. Enter or paste your new API key
  3. Click the save icon

Language

Toggle between Chinese and English using the language switch in the sidebar's user menu. The UI will update immediately.

Build Configuration

The project uses Vite for builds with the following optimizations:

  • Code splitting — AI, Flow, and UI libraries are split into separate chunks
  • Path alias — @/ maps to the project root
  • Tailwind CSS v4 — via the @tailwindcss/vite plugin

FAQ

Frequently asked questions about Musey AI Lab.

General

Is my API key safe?

Yes. Your API key is stored only in your browser's localStorage. It is sent directly to Google's API endpoints and never passes through any third-party server.

Does the app require a backend?

No. Musey AI Lab is a pure frontend application. All AI calls go directly from your browser to Google's APIs.

Will my canvas be saved if I refresh?

Currently, the canvas state (nodes, connections, images) is stored in memory only. Refreshing the page will reset the canvas. History records are persisted in localStorage.

Generation

Why is my generation failing?

Common causes:

  • Invalid API key — check your key in the sidebar settings
  • Quota exceeded — you may have hit the API rate limit
  • Content policy — the prompt may have triggered safety filters

Why does video generation take so long?

Video generation with Veo is computationally intensive and runs asynchronously on Google's servers. Expect 30 seconds to several minutes depending on complexity.

Can I generate multiple images at once?

Yes. Set the "Quantity" parameter in the Image Node to generate 1-4 images per request.

Skills

Why does Change Model need two images?

The skill needs the original scene image (with the person to replace) and a reference image of the target face/person.

What image formats are supported?

PNG, JPEG, and WebP are supported for import. Generated images are returned as PNG.

Deployment

How do I deploy to production?

Run npm run build to create a production build in the /dist folder. Deploy the contents to any static hosting provider (Vercel, Netlify, GitHub Pages, etc.).

Do not embed API keys in environment variables during build. The current user-input model is the secure approach for a frontend-only application.
